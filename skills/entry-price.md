# 买入价分析：多锚点估值 + 分层入场价格带

对 $ARGUMENTS 进行系统化买入价分析，输出明确的分层入场价格带与触发条件。

**支持输入格式**：单个或多个公司，用逗号/顿号/空格分隔。例如：`TSLA` 或 `腾讯, 小米, NVDA`

**定位**：本 skill 回答一个问题——**"什么价格值得买？"** 它不替代 /investment-research 的完整研究（生意/护城河/管理层），而是聚焦估值与价格纪律。如果该公司近期（90天内）已有 research 报告，**必须先读取并复用其结论**，不重复研究。

## 核心理念

> 段永平："买股票就是买公司。对的生意、对的人、对的价格——价格是最后一道关，也是唯一你能完全掌控的一道。"
> 巴菲特："用 4 毛钱买 1 块钱的东西。价格是你付出的，价值是你得到的。"

买入价分析的本质不是"预测股价会跌到哪"，而是**找到风险收益比发生质变的价格拐点**：低于这个价格，即使看错，亏损有限；高于这个价格，必须依赖叙事持续兑现才能赚钱。

## 执行流程

### 第一步：复用已有研究 + 获取实时锚点

1. **检查已有研究**：查找 `reports/{公司名}/` 下最近的 research/checklist/thesis 报告。若存在且在90天内，直接复用其中的：财务数据、护城河评级、风险清单、信息丰富度评级。仅补充增量信息（最新股价、最新季报、重大事件）。
2. **实时数据**：获取当前股价、市值、总股本、PE(TTM)、Forward PE、PB、52周区间。**市值必须手算校验**（股价×总股本）：

```bash
python3 tools/financial_rigor.py verify-market-cap \
  --price {股价} --shares {总股本} --reported {报告市值} --currency {币种}
```

3. 若无近期研究报告，启动后台 Agent 快速收集估值所需最小数据集（TTM EPS、BVPS、每股FCF、近3年增速、分析师目标价分布、独立DCF估值），数据源遵循 `skills/financial-data.md` 双源规范。

### 第二步：建立多锚点估值体系（至少4个独立锚点）

单一估值方法都会错，**用多个相互独立的锚点交叉定位**。每个锚点必须注明方法与假设：

| 锚点 | 方法 | 适用性 |
|------|------|--------|
| **锚点1：基本面三情景**（必做） | `financial_rigor.py three-scenario`，用可验证的盈利做3年推演 | 所有盈利公司 |
| **锚点2：独立DCF** | Alpha Spread / GuruFocus GF Value / 自建反向DCF | 现金流可预测的公司 |
| **锚点3：历史估值分位** | 当前PE/PS/PB处于自身5-10年什么分位（前复权口径） | 上市3年以上 |
| **锚点4：同业横向对比** | 与商业模式最接近的3-5家公司倍数对比 | 有可比公司时 |
| **锚点5：卖方SOTP/目标价光谱** | 目标价最高/最低/中位数，识别分歧来源 | 覆盖充分的公司 |
| **锚点6：期权价值拆解**（叙事股必做） | 现价 = 基本盘价值 + 叙事期权价值，量化拆分 | 高PE叙事驱动股 |

**锚点1 强制工具验算，禁止心算：**

```bash
python3 tools/financial_rigor.py three-scenario \
  --price {股价} --eps {TTM_EPS} --shares {总股本亿} \
  --growth {乐观增速小数} {中性增速小数} {悲观增速小数} \
  --pe {乐观PE} {中性PE} {悲观PE} --years 3 --currency {币种}
```

注意：`--growth` 参数用小数（0.25 = 25%），不是百分数。

**锚点6 期权价值拆解（针对高估值叙事股，如PE>50的公司）：**

```bash
# 基本盘价值 = 可验证净利润 × 传统行业倍数 / 总股本
python3 tools/financial_rigor.py calc --expr "{净利润} * {保守PE} / {总股本}"
```

输出格式：**"现价 $X 中，基本盘值 $Y（Z%），其余 (100-Z)% 是对 {叙事} 的期权定价"**。这是判断"贵不贵"最诚实的方式。

### 第三步：识别估值分歧的核心变量

回答：**多空双方目标价的差距，本质上是在争论什么？**

- 找出1-3个决定估值的核心变量（如：汽车业务盈亏平衡、Robotaxi兑现、毛利率触底时点）
- 对每个变量列出"多方假设 vs 空方假设 vs 当前证据"三栏
- 明确哪些变量是**可跟踪验证的**（季报数据、月度交付），哪些是**不可知的**（技术路线成败、监管走向）

> 芒格："我不做二元期权式的下注，除非赔率极度有利。"——核心变量不可知的比例越高，要求的安全边际越大。

### 第四步：输出分层买入价格带（本 skill 的核心交付）

必须输出以下表格，价格带基于第二步的多锚点交叉定位：

| 策略层 | 入场价格带 | 定位逻辑 | 适合谁 |
|--------|-----------|---------|--------|
| **深度价值层（段永平式）** | ${A}–${B} | 贴近悲观情景/独立DCF下沿，**叙事期权基本免费**，看错也亏损有限 | 严格安全边际投资者，接受可能永远等不到 |
| **平衡层（推荐首选）** | ${C}–${D} | 基本盘价值覆盖大部分成本 + 极小期权费；风险收益比转折点 | 多数价值投资者 |
| **激进层** | ${E}–${F} | 为"方向大概率正确"支付溢价，仍保留一定缓冲 | 对核心叙事有高置信度者 |
| **现价评估** | ${当前价} | 明确判断：现价处于哪一层之上/之内，是否可入场 | — |

**规则**：
1. 每层价格带必须能追溯到第二步的具体锚点，不允许拍脑袋
2. 必须明确说出"推荐首选"是哪一层及理由
3. 若现价已在平衡层以内，直接说"现价可分批买入"；若远高于激进层，直接说"现价不建议入场"——不回避结论
4. 对亏损公司（无EPS），改用 PS/EV-Revenue 情景带替代PE三情景，并注明置信度更低

### 第五步：触发条件（比死守价格更重要）

价格带是静态的，价值是动态的。必须列出：

**下修触发**（出现则整体价格带下移，等更低价）：
- 列出2-4个具体的、可验证的负面事件（如：某季财报毛利率跌破X%、核心业务连续两季负增长）

**上修触发**（出现则价格带上移，或提前入场）：
- 列出2-4个具体的正面验证事件（如：新业务连续两季盈利、核心变量从"不可知"变为"已验证"）

**观察窗口**：列出未来1-2个最关键的验证时点（财报日、产品发布、监管决定）。

### 第六步：仓位与纪律建议

- **分批方案**：给出具体的分批比例建议（如：平衡层上沿建1/3仓，中位加1/3，下沿加1/3）
- **段永平式追问**：如果买入后股价再跌50%，你是恐慌还是加仓？答案取决于你对生意的理解深度，而不是价格本身
- **不为清单**：不因FOMO追高、不借钱、不做空、单一标的不超过组合上限

### 第七步：写入报告

报告写入 `reports/{公司名}/{公司名}-entry-{YYYYMMDD}.md`。多公司时每家独立成文，另在根目录输出对比总表 `reports/多公司买入价对比-{YYYYMMDD}.md`。

## 输出格式要求

1. **报告开头**：当前价格快照（股价/市值/关键倍数，注明数据时点）+ 复用了哪份已有研究
2. **多锚点估值表**：所有锚点及其价格结论汇总在一张表
3. **分层买入价格带表**（核心交付，见第四步格式）
4. **触发条件清单**（下修/上修/观察窗口）
5. 所有涉及计算的数字必须经 `financial_rigor.py` 验算，工具输出嵌入附录
6. 结论明确，不回避：现价能不能买、首选什么价、等什么信号
7. 数据来源标注，估计值注明"估计"
8. 穿插巴菲特/芒格/段永平/李录语录点评
9. **报告结尾声明**：价格带基于报告日数据快照，基本面变化时价格带随之失效；本报告不构成投资建议

## 关键原则

- **价格带必须可追溯**：每个数字都能指回某个估值锚点，拒绝"感觉大概"
- **诚实拆解期权价值**：对叙事股，明确说出现价中多少比例是"还不存在的东西"
- **触发条件优先于价格**：等到价格但基本面已恶化 = 接飞刀；价格未到但核心变量已验证 = 可上修入场价
- **宁可错过，不可做错**：深度价值层可能永远等不到，这是纪律的成本，不是缺陷
- **区分"便宜"与"安全边际"**：便宜的烂生意是价值陷阱；安全边际 = 好生意 + 低于内在价值的价格
